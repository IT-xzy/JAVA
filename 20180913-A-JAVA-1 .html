什么是JDBC 和 JDBC Template？

JDBC（Java Data Base Connectivity,java数据库连接）是一

种用于执行SQL语句的Java API，可以为多种关系数据库

提供统一访问，它由一组用Java语言编写的类和接口组成。

JDBC提供了一种基准，据此可以构建更高级的工具和接口，

使数据库开发人员能够编写数据库应用程序。

JDBC Template就是Spring框架对JDBC进行封装的一个模板（利用依赖注入和反射机制）。

简而言之，JDBC主要作用有三个方面：

建立与数据库的连接

向数据库发送SQL语句

处理数据库返回结果

JDBC Template通俗点可以理解为对原生jdbc进行封装，简化数据库的操作，提高开发效率

JDBC的实现

1.加载驱动

Class.forName("com.mysql.jdbc.Driver")k;

2.获取与数据库的链接

Connection conn = DriverManager.getConnection(url, username, password);

3.获取用于向数据库发送sql语句的statement，创建要发送的SQL语句

Statement st = conn.createStatement();

4.向数据库发sql,并获取代表结果集的resultset

ResultSet rs = st.executeQuery(sql);

5.取出结果集的数据

可以使用 while(rs.next()){};

6.对结果集做相应的处理(增,删,改,查)

7.关闭链接，释放资源

rs.close();

st.close();

conn.close();

JDBC Template的实现

有三种方式：

基于JAVA代码

基于注解

基于XML文件

思路是想通的，以基于XML文件实现为例，关键点是beans.xml文件的配置

JDBC和jdbc template的优缺点对比

JDBC:优点：运行期：快捷、高效

缺点：编辑器：代码量大、繁琐异常处理、不支持数据库跨平台

JDBCTemplate:优点：运行期：高效、内嵌Spring框架中、支

持基于AOP的声明式事务

缺点：必须于Spring框架结合在一起使用、不支持数据库跨

平台、默认没有缓存

关于预编译

1.什么是预编译：

通过Statement对象执行SQL语句时，需要将SQL语句发送给DBMS

由DBMS首先进行编译后再执行。

预编译语句和Statement不同，在创建PreparedStatement

对象时就指定了SQL语句，该语句立即发送给DBMS进

行编译。当该编译语句被执行时，DBMS直接运行编译后的SQL语句，

而不需要像其他SQL语句那样首先将其编译。

2、什么时候使用预编译语句

一般是在需要反复使用一个SQL语句时才使用预编译语句，

预编译语句常常放在一个fo r或者while循环里

面使用，通过反复设置参数从而多次使用该SQL语句

3、为什么使用预编译语句

(1) 防止SQL注入

(2)提高效率

数据库处理一个SQL语句，需要完成解析SQL语句、检查语法和语义以及生成代码，处理时间要比执行语句所需要的时间长。

(3)提高代码的可读性和可维护性

将参数与SQL语句分离出来，这样就可以方便对程序的更改和扩展，同样，也可以减少不必要的错误。
