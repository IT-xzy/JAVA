1.背景介绍

什么是spring boot

随着Spring的功能越来越强，Spring本身变得越来越臃肿。即使只是完成一个发送邮件的小项目也需要配置一堆的配置文件。 SpringBoot简化了Spring应用开发，

2.知识剖析

springboot的优点

1.实现约定大于配置，是一个低配置的应用系统框架。不像Spring那样“地狱般的配置体验”，SpringBoot不需要或极少配置，就能使用大量的功能;

2.提供了内置的Tomcat容器;

3.通过依赖的jar包管理、自动装配技术，容易支持与其他技术体系、工具集成；

4.支持热加载，开发体验好。

3.常见问题

SpringBoot拦截器如何配置？

4.解决方案

SpringBoot通过一个WebMvcConfigurer接口来全局定制化SpringBoot的MVC特性。 开发者可以通过实现WebMvcConfigurer接口来配置应用的MVC全局特性。 全局特性包括：配置拦截器、跨域访问、格式化、URI到视图的映射...

6.扩展思考

URL路径匹配

?：匹配任何单字符

*：匹配0或者任意数量的字符

**：匹配0或者更多的目录

优先匹配原则

如果一个请求有多个@RequestMapping可以匹配，通常是更具体的匹配会作为处理此请求的方法

有通配符的低于没有通配符的

有"**"通配符的低于有"*"通配符的
